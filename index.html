API's and URL's

Below is a list of all APIs associated with a Azure Trigger function:

  1. All APIs start with the same base_url https://streetleaf.azurewebsites.net .
  2. Following the base_url the /api/ prefix is required.
  3. Following the prefix insert any of the triggers below to get the following function: *you must add ? between the function and input:

    4. No Function Input Return
    1 base_url/api/trigger N/A Contains all assets referenced
    2 base_url/api/Login Username
    Password    		 Returns User Login info and token
    3 base_url/api/GetProjectsList token Return Entire Project List including Project IDs
    4 base_url/api/GetProjectsDetails Project ID
    token    		 Returns Details of Specific Projec
    5 base_url/api/GetPoleDetails Pole Number All Static Information of “Pole Number"
    6 base_url/api/GetPolesList Project ID
    token     		Return List of Poles relative to the project
    7 base_url/api/GetPoleModel Model Name
    token    		 ModelInfo
    8 base_url/api/GetCurrentReadings Pole ID Current readings of pole from Pole ID
    9 base_url/api/GetCurrentReadingsForAllPoles Project Id All Current readings of poles from Project ID

What are Functions?

In Azure, Functions are programmed to perform different and specific tasks. Functions often require input parameters to ensure the User is allowed to run said function and to return the correct information to the User.

Functions often need specific Triggers to start

What are Triggers?

In Azure a “trigger” is a function that, in the case of our API, needs a GET or POST method to activate. Once a GET with correct data has been sent to our server, the server will reply to the GET function with the relative data requested.

Here is an example of a https://streetleaf.azurewebsites.net/api/login function reply:

JSON RESPONSE
         
          
          
           [
                {
                    "FirstName": " User Name ",
                    "Token": " Token Key ",
                    "Code": "200",
                    "Status": "Logged in Successfully. Please use this token for rest of data"
                }
            ]

For more information, please visit: Triggers and bindings in Azure Functions | Microsoft Learn

What is SAM?

Serverless Application in Model (SAM) is a AWS open source framework dedicated to building applications. The strength in SAM is that it can intercept requests by the end user and allocate dynamic server use to deliver the response request by the User.

Streetleaf Login

The Login API allows users to retrieve what Token is associated with their account.Tokens are really important and need to be passed at every request except for the Login info

Function Input Return
base_url/api/Login?name=First Name &password= password Username
Password 		Returns User Login info and token
Example:

Input: https://streetleaf.azurewebsites.net/api/Login?Name=name&password=password
Return :
JSON RESPONSE
         
          
          
           [
                {
                    "FirstName": " User Name ",
                    "Token": " Token Key ",
                    "Code": "200",
                    "Status": "Logged in Successfully. Please use this token for rest of data"
                }
            ]

List of Projects

IThe GetProjectsList API allows users to retrieve the list of Projects they will be able to access with their account along with the ID of said projects. ProjectIDs will be used in other API calls to retrieve further information.

Function Input Return
base_url/api/GetProjectsList?token=Token key token Return Entire Project List including Project IDs
Example:

Input: https://streetleaf.azurewebsites.net/api/GetProjectList?token=token
Return :
JSON RESPONSE

              "ProjectID": " Project ID ",,
              "ProjectName": "" Project Name ","

          {
              "ProjectID": " Project ID ",,
